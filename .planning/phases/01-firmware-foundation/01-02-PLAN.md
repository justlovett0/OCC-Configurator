Create the USB descriptor layer for pico-retro — XInput gamepad descriptors for normal mode (subtype 0x01) and CDC serial descriptors for config mode (PID 0xF00F), with the g_config_mode flag switching between them.

Purpose: Defines the USB identity that Windows uses to recognize pico-retro as a gamepad (XInput) or as a serial config device (CDC). The magic vibration constants enable the configurator to trigger config mode entry.
Output: usb_descriptors.h (constants, masks, report struct) and usb_descriptors.c (TinyUSB callbacks).
</objective>

**CDC configuration descriptor** — Identical to pedal (standard ACM CDC with notification + data endpoints).

**String descriptors:**
- Index 0: Language (0x0409 English US)
- Index 1: Manufacturer — "OCC" (same as pedal)
- Index 2: Product — In XInput mode: "Retro Gamepad" (or g_device_name if set and non-empty). In CDC mode: "Retro Gamepad Config"
