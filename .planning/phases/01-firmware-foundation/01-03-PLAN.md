**Core serial I/O helpers** — Copy from pedal: `_serial_read_line()` (reads from CDC until \r or \n, null-terminates), `_serial_write()` (writes string to CDC with tud_task() calls). These are identical across variants.

**PING handler** — Responds with `PONG\r\n`. Identical to pedal.

**GET_CONFIG handler** — Sends multi-line response:
```
DEVTYPE:pico_retro\r\n
CFG:btn0=<pin_buttons[0]>,btn1=<pin_buttons[1]>,...,btn12=<pin_buttons[12]>,pin_lt=<pin_lt>,pin_rt=<pin_rt>,mode_lt=<mode_lt>,mode_rt=<mode_rt>,debounce=<debounce_ms>,device_name=<device_name>,lt_min=<lt_min>,lt_max=<lt_max>,rt_min=<rt_min>,rt_max=<rt_max>,lt_invert=<lt_invert>,rt_invert=<rt_invert>,lt_ema_alpha=<lt_ema_alpha>,rt_ema_alpha=<rt_ema_alpha>\r\n
```

Key names must exactly match the SET command keys below. Use `snprintf` to build the CFG line. All integer values as decimal strings. device_name as-is (string value).

**SET handler** — Parse `SET:key=value`. Supported keys and their handling:

Button pins (13 keys):
- `btn0` through `btn12` → `config->pin_buttons[0]` through `config->pin_buttons[12]`
- Value: integer -1 to 28. -1 = disabled. Validate range.

Trigger pins:
- `pin_lt` → `config->pin_lt`, `pin_rt` → `config->pin_rt`
- Value: integer -1 to 28. -1 = disabled.
- IMPORTANT: If the corresponding mode is INPUT_MODE_ANALOG and the value is not -1, validate that pin is 26, 27, or 28 (ADC-capable). If not, respond `ERR:analog pin must be 26-28\r\n`.

Trigger modes:
- `mode_lt` → `config->mode_lt`, `mode_rt` → `config->mode_rt`
- Value: 0 (digital) or 1 (analog). Validate range 0-1.

Trigger calibration:
- `lt_min` → `config->lt_min`, `lt_max` → `config->lt_max`
- `rt_min` → `config->rt_min`, `rt_max` → `config->rt_max`
- Value: integer 0-4095. Validate range.

Trigger invert:
- `lt_invert` → `config->lt_invert`, `rt_invert` → `config->rt_invert`
- Value: 0 or 1. Validate range.

Trigger EMA:
- `lt_ema_alpha` → `config->lt_ema_alpha`, `rt_ema_alpha` → `config->rt_ema_alpha`
- Value: 0-100. Validate range.

Debounce:
- `debounce` → `config->debounce_ms`
- Value: 0-255. Validate range.

Device name:
- `device_name` → `config->device_name`
- Validate: alphanumeric + space only, max 20 chars. Strip trailing spaces. If invalid chars, respond `ERR:invalid device name\r\n`. Copy validated string with null terminator.
- Follow exact validation pattern from pedal_config_serial.c.

On successful SET: respond `OK\r\n`
On unknown key: respond `ERR:unknown key\r\n`
On invalid value: respond `ERR:<reason>\r\n`

**SAVE handler** — Calls `config_update_checksum(config)` then `config_save(config)`. Responds `OK\r\n`.

**DEFAULTS handler** — Calls `config_set_defaults(config)`. Responds `OK\r\n`.

**REBOOT handler** — Calls `watchdog_reboot(0, 0, 10)` then `while(1) { tight_loop_contents(); }`.

**BOOTSEL handler** — Calls `reset_usb_boot(0, 0)`.

**config_serial_loop()** — Main loop:
```c
void config_serial_loop(retro_config_t *config) {
    while (true) {
        tud_task();
        if (!tud_cdc_connected() || !tud_cdc_available()) {
            sleep_ms(1);
            continue;
        }
        // read line, parse command, dispatch
    }
}
```

Read line into buffer (256 bytes max). Compare against "PING", "GET_CONFIG", "SAVE", "DEFAULTS", "REBOOT", "BOOTSEL". If starts with "SET:", parse key=value. Unknown commands: respond `ERR:unknown command\r\n`.

Do NOT implement SCAN or STOP commands (those are guitar/pedal-specific for button detection — will be added in Phase 2 configurator if needed).
Do NOT implement LED_* commands (pico-retro has no LEDs).
